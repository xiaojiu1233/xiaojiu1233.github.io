购物平台项目文档

一、项目概述

本项目是一个基于Java Web技术的购物平台，分为三端：客户端（用户端）、商家端和后台管理端。系统主要涉及用户管理、商品管理、订单管理、支付、文件上传下载、权限管理等模块，并为后续功能扩展提供了灵活的接口设计和模块化架构。

二、系统架构

​ • 前端技术栈：HTML5, CSS3, JavaScript, Vue.js (或 React)

​ • 后端技术栈：Spring Boot, Spring Security, MyBatis, Redis

​ • 数据库：MySQL (可根据需求选择其他数据库，如MongoDB)

​ • 文件存储：阿里云OSS或本地文件系统

​ • 支付模块：支付宝、微信支付

​ • 消息队列：RabbitMQ或Kafka（可选，用于异步任务处理）

​ • 缓存：Redis（用于缓存用户会话、商品信息等）

三、功能模块

1. 客户端（用户端）

​ • 用户注册与登录：

​ • 支持手机号、邮箱、第三方平台（如微信、QQ）注册。

​ • 用户登录支持账号密码和验证码登录。

​ • 支持忘记密码功能。

​ • 商品展示与浏览：

​ • 支持商品分类查看、搜索、筛选（按价格、评分、销量等）。

​ • 商品详情页展示商品信息、库存、评论等。

​ • 购物车管理：

​ • 支持添加、删除商品至购物车。

​ • 支持修改商品数量、清空购物车。

​ • 订单管理：

​ • 用户可查看历史订单、创建新订单、取消订单。

​ • 查看订单详情、配送地址、支付状态等。

​ • 支付功能：

​ • 支持支付宝、微信支付。

​ • 支付成功后更新订单状态，并提供支付凭证。

​ • 文件上传与下载：

​ • 用户可以上传头像、文件等。

​ • 用户可以下载订单凭证、发票等文件。

​ • 个人中心：

​ • 用户可以查看个人信息、修改密码、查看订单历史等。

2. 商家端

​ • 商品管理：

​ • 商家可管理自己的商品，包括新增、修改、删除、上架、下架等操作。

​ • 支持批量导入商品信息。

​ • 订单管理：

​ • 商家查看订单、确认发货、处理退款等。

​ • 可按订单状态筛选查看。

​ • 权限管理：

​ • 商家可管理店铺的管理员、员工权限。

​ • 不同角色具有不同的访问权限。

​ • 数据统计与报表：

​ • 商家可查看销售报表、订单统计、流量分析等。

3. 后台管理端

​ • 用户管理：

​ • 管理员可以查看、删除用户，重置用户密码，锁定/解锁用户等。

​ • 商品管理：

​ • 管理员可对商家上架的商品进行审核、删除、不合规商品下架。

​ • 订单管理：

​ • 管理员可查看所有用户的订单，进行干预（如取消订单、退款）。

​ • 系统设置：

​ • 管理员可以配置系统的一些全局参数，如支付配置、物流配置等。

​ • 权限管理：

​ • 管理员可以设置不同角色的访问权限（如超级管理员、商家管理员、普通员工）。

四、权限管理

​ • 角色设计：

​ • 客户（Client）：仅能访问客户端功能，如商品浏览、下单、支付等。

​ • 商家管理员（Seller Admin）：管理店铺的商品、订单、用户等。

​ • 系统管理员（System Admin）：管理平台的所有部分，包括用户、商品、订单、权限等。

​ • 权限控制：

​ • 前端权限控制：前端根据用户角色加载不同的页面和功能。

​ • 后端权限控制：后端接口根据用户角色进行权限验证，确保数据安全。
